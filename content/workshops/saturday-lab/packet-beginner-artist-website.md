# Artist Website for Beginners — Saturday Lab

> **Launch edition** of the Digital Lab self-study model · Full cheat sheet: [/workshop/saturday-lab/print/beginner](/workshop/saturday-lab/print/beginner)

**Who this is for:** Artists building or refreshing a public website with no-code tools, Google Docs, or Canva.  
**Today’s goal:** **Structure + one drafted page.** A live URL is a bonus, not required.

**Hub:** [/workshop/saturday-lab](/workshop/saturday-lab)

---

## Remember this

> **A good artist website makes it easy for someone to understand, remember, and contact you.**

**Pattern:** Collect → Organize → Write → Build → Publish → Improve

---

## Owned vs rented presence

| **Owned** | **Rented** |
|-----------|------------|
| Your domain + site you control | Instagram, TikTok, algorithm feeds |
| You decide layout and archive | Platform can change rules overnight |
| Best for grants, curators, long-term | Best for discovery — use both |

**Do this now:** List what you only have on social vs what lives on a site you control.

---

## No account vs account-based

| **No account / low friction** | **Account-based** |
|-------------------------------|-------------------|
| Google Doc sitemap | Squarespace, Wix, Cargo, Framer |
| Canva layout mockup | Notion as site |
| Copy in Notes | Draft in builder |

> No one is blocked without an account. If you can write in a doc, you can start.

---

## Step 1: Audience & goals (10 min)

**Do this now:** Answer in one sentence each:

- Who is your site for? (curators, collectors, community, grant panels)
- What should they do after visiting? (email you, see work, read CV)

---

## Step 2: Site map

```text
Home → Work → About → CV / Press → Contact
```

**Do this now:** Sketch your site map on paper or in a Google Doc. Delete any page that does not answer one clear question.

---

## Step 3: Copy blocks worksheet

Draft these three blocks (even rough):

| Block | Prompt |
|-------|--------|
| **Hero** | Name + one sentence + strongest image |
| **About** | 100-word bio |
| **Contact** | Email + one social link |

**Do this now:** Fill in hero + contact first. About can be draft 2.

---

## Step 4: Choose a platform

| Platform | Best for |
|----------|----------|
| Squarespace | Polished portfolios |
| Wix | Drag-and-drop flexibility |
| Cargo | Contemporary artists |
| Framer | Modern layouts |
| Google Docs + Canva | Before you commit |

---

## Step 5: AI prompts

**Homepage:**

```text
Help me write a clear homepage introduction for my artist website.
I am based in [city]. My work uses [media]. My work explores [themes].
Write 3 versions: very short, warm, and professional/institutional.
```

**Project page:**

```text
Write 150-word and 50-word project descriptions.
Title, year, materials, what the viewer sees, why I made it.
```

**Website audit:**

```text
Act as a curator reviewing my site. What is clear, confusing, missing?
What should I fix first?
```

## Step 6: 30-day maintenance calendar

| Week | Task |
|------|------|
| 1 | Publish or draft homepage |
| 2 | Add or update one project |
| 3 | Check mobile view + contact link |
| 4 | Remove one outdated piece; add one new image |

**Do this now:** Circle one row you will do in the next 7 days.
