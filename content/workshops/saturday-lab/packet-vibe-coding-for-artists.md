## Remember this

## Three lanes (course vocabulary)

| Lane | Stack | When to use |
|------|-------|-------------|
| **Easy** | CodePen + browser + AI chat | First artifact, lowest friction |
| **Structured** | GitHub + VS Code (or Replit) | Files, folders, publish path |
| **AI-assisted** | GitHub + Cursor | Prompt inside editor, review diffs |

Saturday default: start **Easy (L1)** unless you need a URL (**L2**) or have the starter zip (**L3**).

## Starter prompt (full site)

```text
Create a simple one-page artist website using HTML, CSS, and JavaScript.
Include: hero, 3 projects, about, CV highlights, contact. Mobile-friendly.
Mark where I should edit text and images.
```

**Or:** [Download starter zip](/workshops/saturday-lab-starter.zip) for Level 3.

---

## Compare prompt

```text
Show me two versions of this page: one minimal and one more chaotic/experimental.
Same content. Explain what changed.
```

---

## Prompt patterns

**Explain:** `Explain this project like I am a beginner. What does each file do?`

**Edit:** `Change the page to feel more like a poetic net art homepage. Keep it simple.`

**Refine:** `Keep the layout; improve typography, spacing, and contrast.`

**Review:** `Before changing anything, tell me your plan in 3 steps. Wait for my OK.`

---

## Debug prompt

```text
Something is broken. Explain in plain English. Point to the line. Give corrected code.
Do not rewrite everything unless necessary.
```

Ask yourself: What broke? What did I expect? What file? What error? What did I change last?
