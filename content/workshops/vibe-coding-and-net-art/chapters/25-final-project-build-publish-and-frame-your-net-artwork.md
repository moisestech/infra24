This is the capstone where your studies become a coherent work.

Your project does not need to be huge or technically complex. It needs to be intentional.

> **A finished net artwork is not just built. It is also framed.**

## What the final project should do

Your project should:

- use the browser as part of the form
- connect to at least one net art principle from the course
- make one central idea visible or experiential
- feel coherent in title, form, and behavior
- exist as a public-facing or presentation-ready work

It can be small, but it should stand on its own.

## Choose your direction

Your capstone can center one or more approaches:

- browser-space work
- hypertext work
- anti-interface work
- responsive work
- remix work
- identity work
- systems work
- live website work

## Step 1 — Choose a core concept

Write one sentence:

**This artwork explores...**

Examples:

- This artwork explores how a webpage can feel like private memory.
- This artwork explores awkward self-performance through webcam vernacular.
- This artwork explores hidden systems of attention in social platforms.
- This artwork explores glitch as language of mistrust and overload.
- This artwork explores the browser as room rather than page.

### Concept prompts

- What part of the web feels emotionally interesting to you?
- What internet behavior or structure do you want to exaggerate?
- What online experience do you want to reveal?
- What viewer feeling do you want: intimacy, confusion, suspense, repetition, awkwardness?

## Step 2 — Choose your net art principles

Pick 1 to 3 principles:

- browser as medium
- page as space
- link as structure
- interaction as meaning
- glitch as language
- remix as method
- networked identity and presence
- systems as subject
- liveness and time
- publishing as part of artwork

Your project does not need everything. It needs a clear center.

## Step 3 — Choose your build path

### Easy path

**CodePen or simple one-page editor**

Best for single-page concepts and quick experiments.

### Structured path

**GitHub + VS Code**

Best for multi-file work and GitHub Pages publication.

### AI-assisted path

**GitHub + Cursor**

Best for faster iteration with review of generated changes.

### Advanced extension path

**VS Code / Cursor + p5.js / Three.js / CSS experiments**

Best for generative systems and more advanced behavior.

## Step 4 — Build a project plan

Define before building:

- title
- format (one page, linked pages, responsive interaction, remix, system map, live URL piece)
- key behavior
- visual language
- viewer sequence (first, next, last)

### Planning template

- **Title:** `____________`
- **One-sentence concept:** `____________`
- **Net art principles:** `____________`
- **Chosen tool path:** `____________`
- **Key visual mood:** `____________`
- **Key interaction/structure:** `____________`
- **Publishing plan:** `____________`

## Step 5 — Build the artwork

While building, focus on:

- concept clarity
- mood consistency
- one meaningful formal or interaction decision
- visual coherence
- readable relationship between form and idea

Do not try to include everything from the course. Build around one strong center.

## Project checkpoints

### 1) Structure

Can the viewer tell where to begin?

### 2) Meaning

Does web form support the concept?

### 3) Behavior

If interaction/motion exists, does it matter?

### 4) Finish

Does it feel intentional rather than accidental?

### 5) Framing

Do title and statement help the work stand alone?

## Step 6 — Publish or present

Two valid outcomes:

### Option A — Publish live

Examples: GitHub Pages, static host, course-hosted URL, personal site.

### Option B — Prepare as live-ready

If not publishing yet, provide:

- runnable local folder
- screenshots or recording
- clear publishing plan

Live publication is ideal, but a strong live-ready presentation is valid.

## Step 7 — Write the artist statement

Keep it short.

Template:

This work explores `__________`.  
It uses browser/interface/link/network/motion/remix as `__________`.  
I wanted the viewer to feel or notice `__________`.

Example:

This work explores awkward online self-presentation through webcam aesthetics and floating overlays. It uses the browser as staged performance space where platform language becomes portraiture. I wanted the viewer to feel intimacy and discomfort at once.

## Step 8 — Name your influences

Name 1 to 3 references from the course:

- Olia Lialina
- JODI
- Rafaël Rozendaal
- Petra Cortright
- Cory Arcangel
- Eva and Franco Mattes
- Shu Lea Cheang
- Heath Bunting

This places your work in lineage.

## Final deliverables

### Required

- project title
- finished browser artwork
- link or folder
- short artist statement
- chosen influences
- one reflection response

### Optional

- screenshots
- short screen recording
- GitHub repo link
- GitHub Pages link
- process notes
- version history notes

## Final project prompts

Choose one:

- Build a browser work that feels like room, not poster.
- Build a hypertext piece where links create emotional tension.
- Build an anti-interface page with designed resistance.
- Build responsive work where motion changes meaning.
- Build a remix page using internet leftovers as portrait material.
- Build a systems page revealing hidden digital structure.
- Build a live webpage where publication is part of concept.

## Easy / Medium / Advanced

### Easy mode

One-page work with clear concept, strong mood, one meaningful browser decision, title + short statement.

### Medium mode

Multi-section or multi-page work with intentional structure, one meaningful state change, published or ready-to-publish workflow, title + statement + references.

### Advanced mode

Layered sequencing/behavior, stronger formal depth, live or version-aware framing, title + statement + references + process notes.

## Reflection

- What course idea shaped your project most?
- What did the browser allow that another medium would not?
- What remains alive or unresolved in this piece?
- If continuing, what would you deepen next?
