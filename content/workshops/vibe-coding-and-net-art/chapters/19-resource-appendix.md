## 2. Vibecoding tool ladder

### Easy: CodePen

- **Why we teach it:** Fastest way to make and share small web experiments in one browser tab.
- **What students do:** Edit HTML/CSS/JS live, remix templates, learn panels, preview, and fork existing pens.
- **Official docs:** [CodePen documentation](https://blog.codepen.io/docs/)
- **Teaching fit:** Best first environment for absolute beginners and for 10-minute artifact cycles.

### Easy–medium: Browser DevTools

- **Why we teach it:** Inspect, tweak, and understand what the browser is actually rendering.
- **What students do:** Inspect elements, test colors and spacing, view console logs, debug small mistakes.
- **Official docs:** [Chrome DevTools](https://developer.chrome.com/docs/devtools)
- **Teaching fit:** Bridge between visual intuition and technical literacy.

### Medium: GitHub + GitHub Pages

- **Why we teach it:** Versioning, public sharing, and a stable home for web artworks.
- **What students do:** Create a repo, upload `index.html`, publish a live page, iterate with commits.
- **Official docs:** [GitHub Pages quickstart](https://docs.github.com/pages/quickstart)
- **Teaching fit:** Best publishing step after CodePen when students want ownership and permanence.

### Medium: Visual Studio Code

- **Why we teach it:** A fuller editor for multi-file projects and more durable practice.
- **What students do:** Work across `index.html`, `style.css`, `script.js`, assets folders, and extensions.
- **Official docs:** [VS Code getting started](https://code.visualstudio.com/docs/getstarted/getting-started)
- **Teaching fit:** After students outgrow single-file browser editing.

### Advanced: Cursor

- **Why we teach it:** AI-assisted coding for ideation, debugging, and remixing—still requiring taste and direction.
- **What students do:** Prompt for variants, refactor experiments, scaffold layouts, compare outputs critically.
- **Official docs:** [Cursor documentation](https://cursor.com/en-US/docs)
- **Teaching fit:** For students who can **direct** the tool rather than be led by it.

### Advanced: Git / local folders / custom domain

- **Why we teach it:** Authorship discipline, backups, and real publishing practice.
- **What students do:** Track versions, manage folders, deploy to a custom URL, keep a portfolio archive.
- **Official docs:** [GitHub: Get started with Git](https://docs.github.com/en/get-started)
- **Teaching fit:** Final stage for an ongoing browser-based practice.

## 3. Net art principles

### The browser is a medium, not just a container

**Meaning:** A webpage can be artwork in its own right. Layout, timing, linking, and rendering are part of the form.

**Artists / works:** Olia Lialina, Rafaël Rozendaal

- **Easy mode:** Change a blank page into a tiny artwork using only text and background color.
- **Medium mode:** Build a one-page visual site with deliberate composition and mood.
- **Advanced mode:** Design a page whose structure changes the viewer’s emotional reading.

### Hypertext creates meaning

**Meaning:** Links do not only navigate; they shape narrative, suspense, and interpretation.

**Artists / works:** Olia Lialina

- **Easy mode:** Two linked pages with one tonal shift.
- **Medium mode:** A branching mini-story with 3–4 pages.
- **Advanced mode:** A nonlinear sequence where users can get lost on purpose.

### Interface can be expressive or hostile

**Meaning:** Net art can resist smooth usability and use confusion as aesthetic strategy.

**Artists / works:** JODI, Mark Napier

- **Easy mode:** Awkward spacing and broken-looking text.
- **Medium mode:** An anti-interface page that feels unstable but still explorable.
- **Advanced mode:** A deliberately deceptive or disorienting interface with behavior changes.

### Interaction is part of the artwork

**Meaning:** Hover, click, scroll, and time-based change can be the core event.

**Artists / works:** Rafaël Rozendaal, Mark Napier

- **Easy mode:** Hover changes one word or color.
- **Medium mode:** Click changes the entire mood of the page.
- **Advanced mode:** Interaction as choreography with multiple states.

### Glitch and distortion can be language

**Meaning:** Error, overload, repetition, and brokenness can communicate feeling and critique.

**Artists / works:** JODI

- **Easy mode:** A glitch text field.
- **Medium mode:** Layer repeated symbols, broken grids, and visual noise.
- **Advanced mode:** Distortion combined with animation or interaction.

### The network is social, political, and infrastructural

**Meaning:** The web is built from protocols, borders, labor, identity, and access systems.

**Artists / works:** Heath Bunting, Shu Lea Cheang, Eva & Franco Mattes

- **Easy mode:** Map one hidden system affecting your life.
- **Medium mode:** A page about surveillance, labor, or online identity.
- **Advanced mode:** Links, diagrams, and interactions critiquing a system.

### Remix is an artistic method

**Meaning:** Found images, screenshots, memes, and text fragments can become composition.

**Artists / works:** Cory Arcangel, Eva & Franco Mattes, Petra Cortright

- **Easy mode:** A collage page from three found fragments.
- **Medium mode:** A remix page that comments on internet culture.
- **Advanced mode:** Remix plus scripting or responsive behavior for a stronger conceptual frame.

### Digital selfhood is staged

**Meaning:** Homepages, avatars, filters, webcams, and feeds shape how identity is performed online.

**Artists / works:** Petra Cortright, Shu Lea Cheang, Eva & Franco Mattes

- **Easy mode:** A self-portrait homepage.
- **Medium mode:** A platform-self page that exaggerates taste or persona.
- **Advanced mode:** A work that reveals and conceals identity at the same time.
