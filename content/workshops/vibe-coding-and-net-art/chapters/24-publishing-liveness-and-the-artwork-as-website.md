A net artwork is not only something you make.

It is also something you publish.  
It has a URL.  
It has a host.  
It loads in real time.  
It may change over time.  
It may break.  
It may survive as a live site, a recording, a restaging, or a version trail.

That means publishing is not an afterthought.

> **In net art, the website can be the artwork, and going live can be part of the form.**

## Why this matters

A screenshot of a webpage is not the same as the webpage.

A recording of an interaction is not the same as a live encounter.

A PDF about a browser piece is not the same as a functioning URL.

GitHub Pages is a strong beginner model because it publishes HTML/CSS/JS directly from a repository into a live website. Tate's net art preservation writing reinforces that keeping web-based works online and functional is part of the curatorial challenge, not a side task.

## Working definitions

For this workshop:

- **Publishing** means making a browser work available at a real URL.
- **Liveness** means the work exists as an active encounter, not only documentation.
- **Versioning** means the work can exist across saved states and revisions over time.

This can include hosted experiments, personal sites, GitHub Pages deployments, timed state changes, and revision histories that matter to meaning.

## Key ideas

### 1) A website is not just a file

A local folder and a live site are related but not identical.

Once published, the work has a URL, enters public circulation, and can be visited in different browsers and contexts.

### 2) The URL can be part of the artwork

Title, domain, and path all affect framing.

Many internet artworks are remembered by address. Rafaël Rozendaal is a core reference for this: the site itself is often the complete piece.

### 3) Live behavior differs from documentation

A live site can depend on timing, interaction, and browser context.

Documentation preserves evidence, but it is not identical to use of the live work.

### 4) Publishing creates public presence

When a page goes live, it shifts from private study to public object.

Even a minimal page can become destination, reference, and shareable artwork.

### 5) Versioning is part of digital form

Drafts, commits, branches, revisions, and restagings can become part of how a work is read.

### 6) Liveness includes fragility

A live work can disappear, fail, drift, lose dependencies, or require maintenance.

That fragility is part of what makes web-native art distinct.

## Documentation vs live web artwork

### Documentation

- screenshots
- recordings
- descriptions
- after-the-fact summaries
- evidence of the work

### Live web artwork

- real URL
- timing
- loading
- interaction
- browser context
- public access
- versioned/changing states

Neither is inherently better; they do different jobs. In net art, live state often carries core meaning.

## Questions to ask

- Is the site itself the work, or documenting another work?
- What changes when visited live?
- Does the URL matter?
- Does timing matter?
- Is the work stable, updated, or fragile?
- What does publishing add to meaning?
- Could screenshots alone fully convey it?

## Mini artifact

### Live Site Study

Make a one-page browser work and treat publishing as part of the concept.

Choose one direction:

- a page with meaningful title and URL path
- a page that feels different once live
- a page with one timed behavior/state shift
- a page where version history matters
- a page framed as a small public artwork

Your goal is not polished portfolio launch. Your goal is one browser work where publication itself matters.

### Easy mode

Create a small page with a title, clear filename, and one public-facing framing sentence.

### Medium mode

Publish a simple HTML/CSS work using GitHub Pages (or equivalent static host).

### Advanced mode

Publish a work where timing, revision, or live-state behavior is intentionally part of the piece.

## Reflection

- What changed when you treated the page as live site rather than class exercise?
- Did publishing feel like documentation, performance, or completion?
- What mattered most: URL, timing, public access, or version history?
- How does a live website change what counts as artwork?
